# Triple Language Mini Search Engine

I came up with and built this project to demonstrate my equal proficiency with Java, C, and C++. 

Each language does the same thing, and you can select which one you want to use for the logic and algorithm in the CLI. 

This is the logic behind the code:

1. Read the `.txt` files in `sample_data/`
2. Tokenize all of the words
3. Build a frequency index of every word
4. Let the user search for a word
5. Return the files ranked by the word's frequency

Example:

> Search: apple
1. notes.txt — 14 matches
2. article.txt — 7 matches
3. journal.txt — 2 matches
